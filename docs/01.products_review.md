# 상품 목록 페이지 리뷰

## 고객사 요구사항

1. 주요 타겟인 20~30대의 젊은 세대가 주로 모바일을 사용하기 때문에 모바일 쇼핑이 불편하면 구매 포기 가능성이 높아 모바일 환경에서의 사용성을 중요하게 생각합니다.
2. 상품의 사진을 강조해서 신발의 사진이 상품명보다 더 중요하게 잘 보여야 합니다.
3. 장바구니 실시간 연동으로 장바구니에 상품을 추가할 때, 장바구니에 몇 개 담겼는지 실시간으로 표시되어야 합니다.
4. 작업 완료 후 테스트용 배포 URL을 공유하고, 내부 피드백을 수렴해야 합니다.

## 요구사항에 따른 기능 동작 확인

각 기능은 고객사 요구사항 문서 및 피그마 시안을 기반으로 검토되었습니다.

### 1. 모바일 환경 구성
예상 동작 : 20~30대 젊은 세대의 모바일 사용성을 고려하여, 다양한 화면 크기에서 2열 카드 레이아웃으로 상품 목록이 최적화되어 표시되어야 합니다.

### 2. 신발 상품 사진
예상 동작 : 신발 사진이 피그마 시안처럼 명확하고 선명하게, 디자인 의도에 맞춰 잘 보여야 합니다.
확인 필요 : 
    실제 신발 사진이 Shoe1, Shoe2 와 같은 플레이스홀더 텍스트가 표시된 곳에 잘 들어가는 지 확인해야 합니다. 
    products.js 에 실제 이미지 URL을 연결한 후, 실제 상품 이미지가 피그마 시안과 동일한 품질, 해상도, 비율, 선명도로 잘 표시되는지 확인해야 합니다.
    
### 3. 장바구니 담기 기능
예상 동작 : '담기' 버튼을 누르면 장바구니에 상품이 추가되고, 화면 상단의 장바구니 글자 옆 숫자가 실시간으로 증가해야 합니다. 
확인 필요 : 장바구니 글자가 아닌 장바구니 이모지가 필요합니다.

### 4. 배포 URL

예상 동작 : 고객사가 작업 결과물을 직접 확인할 수 잇는 테스트 배포 URL이 제공되어야 합니다.
확인 필요 : 고객사가 실제 구현된 페이지를 확인할 수 있는 웹 애플리케이션의 유효한 배포 URL (예: Netlify, Vercel 등에 배포된 실제 주소)을 확보해야 합니다.

### 5. 무한 스크롤 기능
예상 동작 : 사용자가 스크롤을 내릴수록 다음 상품이 자동으로 로드되어야 합니다. 성능 최적화를 위한 Lazy Loading 구조가 고려되어야 합니다.
확인 필요 : 구현되었음을 확인하려면, 페이지 하단으로 스크롤 시 새로운 상품들이 자동으로 로드되는지, 그리고 로드 시 페이지 끊김 없이 부드럽게 작동하는지에 대한 실제 테스트 결과가 필요합니다.
