# 장바구니 프로젝트

## 프로젝트 개요

  - **프로젝트 목표**: 사용자들이 여러 상품을 장바구니에 담아 한 번에 확인하고, 정확한 총 금액을 계산하여 편리하게 결제할 수 있는 페이지를 제공합니다.
  - **주요 기능**: 상품 목록 표시, 수량 조절, 상품 금액 및 배송비 자동 계산, 100,000원 이상 구매 시 배송비 무료 로직, 페이지 간 네비게이션 기능 등을 포함합니다.
  - **접속할 수 있는 페이지 url**: (개발 서버 실행 후 제공 가능)
  - **참여한 팀원**: 이정현 (ezeonon)

## 개발 환경

  - **사용한 프레임워크, 라이브러리, 언어**: React 18+, styled-components, react-router-dom, React Context API, JavaScript ES6+
  - **설치 및 실행 방법**: `npm install` 후 `npm start` 명령어로 프로젝트를 실행합니다.

## 설치 및 실행 방법

  - **프로젝트를 실행하기 위한 명령어**:
    ```bash
    npm start
    ```

## 컴포넌트 구조

  - **주요 컴포넌트의 역할과 구조**:
      - `App.js`: 애플리케이션의 최상위 컴포넌트로, `CartProvider`를 사용하여 장바구니 상태를 전역적으로 관리하고 라우팅을 설정합니다.
      - `CartProvider`: 장바구니에 담긴 상품 목록을 관리하는 Context입니다. 상품 추가/삭제, 수량 조절 등의 로직을 제공합니다.
      - `CartScreen.js`: 장바구니 페이지의 전체 레이아웃을 담당하는 메인 화면 컴포넌트입니다.
      - `CartItem.js`: 장바구니에 담긴 상품 하나를 나타내는 컴포넌트입니다.
      - `CartSummary.js`: 페이지 하단에 위치하며, 결제 요약 정보를 계산하고 표시합니다.

## 테스트 방법과 스토리북 컴포넌트 링크

  - **테스트를 진행할 수 있는 절차**: `npm start` 명령어로 개발 서버를 실행한 후, 브라우저에서 수동으로 다음 항목들을 확인합니다.
    1.  상품 수량 조절(`-`, `+`) 버튼을 눌렀을 때 총 금액이 올바르게 변하는지.
    2.  상품 수량이 0이 될 경우 상품이 삭제되는지.
    3.  총액에 따른 배송비 계산 로직이 올바른지.
    4.  뒤로가기/결제하기 버튼이 올바른 페이지로 이동하는지.
  - **문서화된 스토리북 페이지**: (제가 직접 생성 및 접속할 수 없으므로, 해당 항목은 비워두겠습니다.)

## 유의사항 및 알려진 이슈

  - **프로젝트에서 발생할 수 있는 문제점이나 유의해야 할 사항**:
      - 장바구니 데이터는 애플리케이션 실행 중에만 메모리 상에 유지됩니다. 서버를 재시작하면 초기화됩니다.
      - 상품별 재고 제한 기능은 현재 구현되지 않았습니다.
