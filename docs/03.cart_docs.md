# 장바구니 프로젝트

## 프로젝트 개요

  - **프로젝트 목표**: 사용자들이 여러 상품을 장바구니에 담아 한 번에 확인하고, 정확한 총 금액을 계산하여 편리하게 결제할 수 있는 페이지를 제공하는 것을 목표로 합니다.
  - **주요 기능**: 상품 목록 표시, 수량 조절, 상품 금액 및 배송비 자동 계산, 100,000원 이상 구매 시 배송비 무료 로직, 페이지 간 네비게이션 기능 등을 포함합니다.
  - **참여한 팀원**: 이정현 (ezeonon)

## 개발 환경

  - **프레임워크**: React 18+
  - **라이브러리**: `styled-components`, `react-router-dom`, React Context API
  - **언어**: JavaScript, HTML5, CSS3

## 설치 및 실행 방법

프로젝트를 로컬 환경에서 실행하기 위한 방법은 다음과 같습니다.

1.  **프로젝트 생성**: `npx create-react-app my-payment-app` (만약 프로젝트가 없다면)
2.  **필요 라이브러리 설치**:
    ```bash
    npm install styled-components react-router-dom
    ```
3.  **개발 서버 실행**:
    ```bash
    npm start
    ```
    이 명령어를 실행하면 웹 브라우저에서 장바구니 페이지를 확인할 수 있습니다.

## 컴포넌트 구조

프로젝트는 역할에 따라 다음과 같은 주요 컴포넌트들로 구성됩니다.

  - **`App.js`**: 애플리케이션의 최상위 컴포넌트로, `CartProvider`를 사용하여 장바구니 상태를 전역적으로 관리하고 라우팅을 설정합니다.
  - **`CartProvider`**: 장바구니에 담긴 상품 목록을 관리하는 Context입니다. 상품 추가/삭제, 수량 조절 등의 로직을 제공합니다.
  - **`CartScreen.js`**: 장바구니 페이지의 전체 레이아웃을 담당하는 메인 화면 컴포넌트입니다.
  - **`CartHeader.js`**: 페이지 상단에 위치하며, 뒤로 가기 버튼과 동적으로 업데이트되는 상품 개수 정보를 표시합니다.
  - **`CartItem.js`**: 장바구니에 담긴 상품 하나를 나타내는 컴포넌트로, 이미지, 브랜드명, 가격, 수량 및 수량 조절 버튼을 포함합니다.
  - **`CartSummary.js`**: 페이지 하단에 위치하며, 상품 금액, 배송비, 총 금액을 계산하고 표시하는 결제 요약 컴포넌트입니다.

## 테스트 방법

다음과 같은 테스트 케이스를 통해 장바구니 페이지의 주요 기능을 확인할 수 있습니다.

  - **초기 상태 확인**: 페이지 접속 시 장바구니가 비어있는지, '장바구니가 비어있습니다.'와 같은 안내 문구가 표시되는지 확인합니다.
  - **상품 추가 및 수량 조절**: 장바구니에 상품을 추가한 후, `+`와 `-` 버튼을 눌러 수량이 올바르게 변하고 총 금액이 실시간으로 업데이트되는지 확인합니다.
  - **상품 삭제**: `-` 버튼을 계속 눌러 수량이 0이 되었을 때 해당 상품이 장바구니 목록에서 자동으로 사라지는지 확인합니다.
  - **배송비 계산**:
      - 총 상품 금액이 90,000원일 때, 배송비가 3,000원으로 계산되는지 확인합니다.
      - 총 상품 금액이 105,000원일 때, 배송비가 0원으로 표시되는지 확인합니다.
  - **페이지 이동**: '뒤로 가기' 버튼 클릭 시 상품 목록 페이지로, '결제하기' 버튼 클릭 시 카드 선택 페이지로 올바르게 이동하는지 확인합니다.

## 유의사항 및 알려진 이슈

  - **데이터 영속성**: 현재 장바구니 데이터는 애플리케이션이 실행되는 동안에만 유지됩니다. 서버를 재시작하거나 브라우저를 새로고침하면 장바구니의 상품 목록은 초기화됩니다.
  - **향후 확장 계획**:
      - 상품별 재고를 고려하여 수량을 제한하는 기능이 필요할 수 있습니다.
      - 진짜 결제 시스템과 연동하기 위해서는 `결제하기` 버튼에 실제 결제 로직을 추가해야 합니다.
  - **페이지 연동**: `뒤로 가기` 및 `결제하기` 버튼은 이전에 구현한 페이지들과 연동되어야 하며, 이는 라우팅 설정을 통해 연결될 예정입니다.
