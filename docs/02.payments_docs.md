# 결제 모듈 프로젝트

## 프로젝트 개요

- 프로젝트 목표: 이 결제 모듈은 사용자가 카드를 등록하고 관리하며, 결제에 필요한 정보를 안전하게 입력할 수 있도록 돕는 웹 컴포넌트입니다.
- 주요기능: 카드 목록 조회, 카드 등록, 실시간 입력 폼 유효성 검사, 민감 정보(카드 번호, CVC, 비밀번호) 마스킹 처리, 화면 간 네비게이션 기능 등을 포함합니다.
- 참여한 팀원: 이정현 (ezeonon)


## 개발 환경

- 프레임워크: React 18+
- 라이브러리: react-router-dom (6.x 이상) styled-components, React Context API
- 언어: JavaScript ES6+, HTML5, CSS3

## 설치 및 실행 방법

프로젝트를 로컬에서 실행하기 위한 설치 방법과 명령어

1. 프로젝트 초기 설정:
```
npx create-react-app my-payment-app
cd my-payment-app
```
2. 필수 라이브러리 설치:
```
npm install react-router-dom styled-components
```
3. 코드 파일 배치: src 폴더에 제공된 코드 파일들을 올바르게 복사합니다.
4. 개발 서버 실행: 이 명령어를 실행하면 http://localhost:3000에서 프로젝트를 확인할 수 있습니다.
```
npm start
```


## 컴포넌트 구조

주요 컴포넌트들의 구조와 역할
- App.js: 애플리케이션의 최상위 컴포넌트로, react-router-dom을 사용하여 화면 간 라우팅을 관리합니다.

- CardListScreen.js: 보유카드 목록을 보여주는 화면 컴포넌트입니다. 등록된 카드와 카드 추가 버튼을 표시하며, CardContext에서 카드 정보를 가져와 렌더링합니다.

- AddCardScreen.js: 새로운 카드를 등록하기 위한 입력 폼 화면입니다. 카드 정보 입력 및 유효성 검사, 실시간 카드 미리보기 기능을 담당합니다.

- CardPreview.js: 입력된 정보를 바탕으로 실시간으로 업데이트되는 카드 미리보기 UI 컴포넌트입니다.

- CardContext.js: 등록된 카드들의 상태를 전역적으로 관리하는 Context API입니다. registeredCards 배열과 addCard 함수를 제공하여 모든 컴포넌트가 카드 상태를 공유할 수 있도록 합니다.

- cardUtils.js: 카드 번호 포맷팅, 만료일 유효성 검사, 카드 번호 마스킹 등 카드 관련 로직을 모아둔 유틸리티 파일입니다.

## 테스트 방법

테스트 케이스 및 실행 방법
- 개발 서버 시작:
```
npm start
```
- 테스트 케이스 :
    - [초기 화면] 브라우저 접속 시, 등록된 카드가 없고 "새로운 카드를 등록해주세요."라는 문구만 표시되는지 확인합니다.

    - [카드 추가] "새로운 카드를 등록해주세요." 버튼 클릭 시, 카드 추가 화면으로 이동하는지 확인합니다.

    - [입력 유효성] 카드번호, 만료일, CVC 등 각 입력 필드에 올바르지 않은 값(숫자 외 문자, 길이 초과) 입력 시, 에러 없이 제한되는지 확인합니다.

    - [버튼 활성화] 모든 필수 입력 필드가 유효하게 입력되었을 때만 "작성 완료" 버튼이 활성화되는지 확인합니다.

    - [카드 등록] 카드 정보를 모두 입력하고 "작성 완료" 버튼을 클릭했을 때, 보유카드 화면으로 돌아와 방금 등록한 카드가 목록에 추가되었는지 확인합니다.

    - [다중 카드] 첫 번째 카드 등록 후, "다른 카드 추가" 버튼을 눌러 두 번째 카드를 등록했을 때, 두 카드가 모두 목록에 표시되는지 확인합니다.

## 유의사항 및 알려진 이슈

- 데이터 영속성: 현재 카드 정보는 애플리케이션이 실행되는 동안에만 메모리 상에 유지됩니다. 개발 서버를 재시작하거나 웹 브라우저를 닫으면 저장된 카드 정보는 초기화됩니다. 영구적인 데이터 저장이 필요할 경우, 별도의 API 서버나 로컬 스토리지를 연동해야 합니다.

- 민감 정보 보안: 현재 구현은 클라이언트 사이드에서 민감 정보(CVC, 비밀번호 등)를 마스킹하는 것에 초점을 맞췄습니다. 실제 서비스에서는 이러한 민감 정보가 네트워크를 통해 전송될 때 암호화가 필수적이며, 백엔드 서버에서 안전하게 처리되어야 합니다.

- 반응형 디자인: 기본적인 반응형 디자인을 적용하여 모바일, 태블릿, 데스크탑 환경에서 보기 좋게 만들었습니다. 그러나 모든 디바이스 및 브라우저 환경에서 완벽한 호환성을 보장하려면 추가적인 디자인 및 CSS 최적화 작업이 필요할 수 있습니다.
