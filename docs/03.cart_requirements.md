# 장바구니 페이지 요구사항 분석 및 기능 목록

## 📌 기본 정보
### 프로젝트명: 
패션 쇼핑몰 장바구니 페이지

### 사용하게 될 기술: 
React (프론트엔드), styled-components, React Context API

### 문서 작성자: 
이정현 (ezeonon)

## 📝 고객 요구사항 정리

1.  **총 금액의 정확성**: 상품 가격, 수량, 배송비를 기반으로 총 결제 금액을 정확하게 계산하고 표시해야 합니다.
2.  **배송비 표시**:
    -   총 상품 금액이 100,000원 미만일 경우 배송비(3,000원)를 표시합니다.
    -   총 상품 금액이 100,000원 이상일 경우 배송비 무료임을 명확하게 표시합니다.
3.  **상품 목록 표시**:
    -   장바구니에 담긴 상품들을 시안에 맞춰 목록 형태로 표시합니다.
    -   한 화면에 3개 이상의 상품이 있을 경우 스크롤이 가능해야 합니다.
4.  **상품 수량 조절**:
    -   상품별로 수량을 늘리거나 줄일 수 있는 버튼(`+`, `-`)을 제공합니다.
    -   수량이 0이 될 경우 해당 상품을 장바구니에서 자동으로 삭제합니다.
5.  **동적 정보 표시**:
    -   페이지 상단에 장바구니에 담긴 전체 상품 개수를 `현재 n개의 상품이 담겨있습니다.` 형식으로 동적으로 표시합니다.
6.  **페이지 이동**:
    -   상단 뒤로 가기 버튼을 누르면 상품 목록 페이지로 이동합니다.
    -   하단 "결제하기" 버튼을 누르면 카드 선택 페이지로 이동합니다.
7.  **디자인 적용**: 제공된 피그마 시안의 레이아웃, 크기, 위치 정보 등을 충실히 반영하여 구현합니다.

## 📋 기능 목록
본 장바구니 페이지는 React 기반의 컴포넌트 아키텍처로 구현될 예정이며, 주요 기능별 컴포넌트 및 필요한 기능들은 다음과 같습니다.

### **`<App />`** (전체 애플리케이션 라우팅 및 상태 관리)
-   `CardProvider`와 유사한 `CartProvider`를 사용하여 장바구니 상태를 전역적으로 관리합니다.
-   라우팅을 통해 `CartScreen`과 다른 페이지들 간의 이동을 담당합니다.

### **`CartProvider`** (장바구니 상태 관리 Context)
-   **상태**: 장바구니에 담긴 상품 정보(id, name, price, quantity, image 등)를 배열 형태로 저장합니다.
-   **기능**: 상품 추가/삭제, 수량 증가/감소 등 장바구니 상태를 변경하는 함수를 제공합니다.

### **`<CartScreen />`** (장바구니 페이지 화면)
-   **UI 요소**:
    -   상단 헤더 (뒤로 가기 버튼, 제목, 상품 개수)
    -   스크롤 가능한 상품 목록
    -   결제 요약 정보 (상품 금액, 배송비, 총 금액)
    -   "결제하기" 버튼
-   **기능**:
    -   `CartProvider`에서 장바구니 상태를 가져와 화면을 렌더링합니다.
    -   각종 금액을 계산하여 `CartSummary` 컴포넌트에 전달합니다.

### **`<CartItem />`** (상품 목록의 개별 아이템 컴포넌트)
-   **UI 요소**: 상품 이미지, 브랜드명, 가격, 수량 및 수량 조절 버튼(`-`, `+`)
-   **기능**:
    -   수량 조절 버튼 클릭 시, `CartProvider`의 상태를 업데이트합니다.
    -   수량이 0이 되면 `CartProvider`의 상품 삭제 함수를 호출합니다.

### **`<CartSummary />`** (결제 요약 정보 컴포넌트)
-   **UI 요소**: "상품 금액", "배송비", "총 금액"을 표시하는 영역 및 구분선, "결제하기" 버튼
-   **기능**:
    -   `CartProvider`에서 가져온 상품 목록을 기반으로 총 상품 금액을 계산합니다.
    -   총 상품 금액에 따라 배송비를 계산하고, 최종 총 금액을 산출하여 표시합니다.
    -   "결제하기" 버튼 클릭 시 카드 선택 페이지로 이동합니다.
