# 장바구니 페이지 요구사항 분석 및 기능 목록

## 📌 기본 정보
### 프로젝트명: 
패션 쇼핑몰 장바구니 페이지

### 사용하게 될 기술: 
React (프론트엔드), styled-components, React Context API

### 문서 작성자: 
이정현 (ezeonon)

## 📝 고객 요구사항 정리

1.  **총 금액의 정확성**: 상품 가격, 수량, 배송비를 기반으로 총 결제 금액을 정확하게 계산하고 표시해야 합니다.
2.  **배송비 표시**:
    -   총 상품 금액이 100,000원 미만일 경우 배송비(3,000원)를 표시합니다.
    -   총 상품 금액이 100,000원 이상일 경우 배송비 무료임을 명확하게 표시합니다.
3.  **상품 목록 표시**:
    -   장바구니에 담긴 상품들을 시안에 맞춰 목록 형태로 표시합니다.
    -   한 화면에 3개 이상의 상품이 있을 경우 스크롤이 가능해야 합니다.
4.  **상품 수량 조절**:
    -   상품별로 수량을 늘리거나 줄일 수 있는 버튼(`+`, `-`)을 제공합니다.
    -   수량이 0이 될 경우 해당 상품을 장바구니에서 자동으로 삭제합니다.
5.  **동적 정보 표시**:
    -   페이지 상단에 장바구니에 담긴 전체 상품 개수를 `현재 n개의 상품이 담겨있습니다.` 형식으로 동적으로 표시합니다.
6.  **페이지 이동**:
    -   상단 뒤로 가기 버튼을 누르면 상품 목록 페이지로 이동합니다.
    -   하단 "결제하기" 버튼을 누르면 카드 선택 페이지로 이동합니다.
7.  **디자인 적용**: 제공된 피그마 시안의 레이아웃, 크기, 위치 정보 등을 충실히 반영하여 구현합니다.

## 📋 기능 목록
본 장바구니 페이지는 React 기반의 컴포넌트 아키텍처로 구현될 예정이며, 주요 기능별 컴포넌트 및 필요한 기능들은 다음과 같습니다.

### **`<App />`** (전체 애플리케이션 라우팅 및 상태 관리)
-   `CardProvider`와 유사한 `CartProvider`를 사용하여 장바구니 상태를 전역적으로 관리합니다.
-   라우팅을 통해 `CartScreen`과 다른 페이지들 간의 이동을 담당합니다.

### **`CartProvider`** (장바구니 상태 관리 Context)
-   **상태**: 장바구니에 담긴 상품 정보(id, name, price, quantity, image 등)를 배열 형태로 저장합니다.
-   **기능**: 상품 추가/삭제, 수량 증가/감소 등 장바구니 상태를 변경하는 함수를 제공합니다.

### **`<CartScreen />`** (장바구니 페이지 화면)
-   **UI 요소**:
    -   상단 헤더 (뒤로 가기 버튼, 제목, 상품 개수)
    -   스크롤 가능한 상품 목록
    -   결제 요약 정보 (상품 금액, 배송비, 총 금액)
    -   "결제하기" 버튼
-   **기능**:
    -   `CartProvider`에서 장바구니 상태를 가져와 화면을 렌더링합니다.
    -   각종 금액을 계산하여 `CartSummary` 컴포넌트에 전달합니다.

### **`<CartItem />`** (상품 목록의 개별 아이템 컴포넌트)
-   **UI 요소**: 상품 이미지, 브랜드명, 가격, 수량 및 수량 조절 버튼(`-`, `+`)
-   **기능**:
    -   수량 조절 버튼 클릭 시, `CartProvider`의 상태를 업데이트합니다.
    -   수량이 0이 되면 `CartProvider`의 상품 삭제 함수를 호출합니다.

### **`<CartSummary />`** (결제 요약 정보 컴포넌트)
-   **UI 요소**: "상품 금액", "배송비", "총 금액"을 표시하는 영역 및 구분선, "결제하기" 버튼
-   **기능**:
    -   `CartProvider`에서 가져온 상품 목록을 기반으로 총 상품 금액을 계산합니다.
    -   총 상품 금액에 따라 배송비를 계산하고, 최종 총 금액을 산출하여 표시합니다.
    -   "결제하기" 버튼 클릭 시 카드 선택 페이지로 이동합니다.
 
### **기능 구현을 위한 세부 작업 및 예외처리**

#### **2-1. `<CartScreen />` (장바구니 페이지 화면)**

* **주요 역할**: 장바구니 페이지의 전체 레이아웃을 구성하고, 상품 목록과 결제 요약 컴포넌트를 통합하여 렌더링합니다.
* **사용 컴포넌트**: `CartHeader`, `CartItemList`, `CartSummary`
* **세부 작업**:
    1.  `CartProvider`에서 장바구니 상품 목록(배열) 상태를 가져옵니다.
    2.  상품 목록이 비어있을 경우, "장바구니가 비어있습니다."와 같은 안내 메시지를 표시합니다.
    3.  상품 목록이 있을 경우, 각 상품을 `<CartItem />` 컴포넌트로 렌더링합니다.
* **예외처리**:
    -   장바구니 상태가 비어있을 경우, 상품 목록 대신 적절한 안내 문구와 UI를 표시합니다.

#### **2-2. `<CartHeader />` (페이지 상단 헤더)**

* **주요 역할**: 페이지 제목과 장바구니에 담긴 상품 개수, 뒤로가기 버튼을 표시합니다.
* **사용 컴포넌트**: `BackButton`
* **세부 작업**:
    1.  `CartProvider`에서 장바구니 상품 목록 상태를 가져와 `총 상품 개수`를 계산합니다.
    2.  "현재 n개의 상품이 담겨있습니다."와 같이 동적으로 상품 개수를 표시합니다.
    3.  뒤로 가기 버튼 클릭 시, 라우팅을 통해 상품 목록 페이지로 이동합니다.
* **예외처리**:
    -   상품 개수가 0일 경우 "현재 0개의 상품이 담겨있습니다." 또는 "장바구니가 비어있습니다."와 같이 상태를 명확히 표시합니다.

#### **2-3. `<CartItem />` (상품 목록의 개별 아이템)**

* **주요 역할**: 상품 하나에 대한 정보를 표시하고 수량 조절 기능을 제공합니다.
* **사용 컴포넌트**: `ProductImage`, `ProductInfo`, `QuantityControls` (`-`, `+` 버튼)
* **세부 작업**:
    1.  상품의 이미지, 브랜드명, 가격을 표시합니다.
    2.  `+` 버튼 클릭 시 `CartProvider`의 수량 증가 함수를 호출하여 상태를 업데이트합니다.
    3.  `-` 버튼 클릭 시 `CartProvider`의 수량 감소 함수를 호출하여 상태를 업데이트합니다.
* **예외처리**:
    -   수량 감소 시, 상품 수량이 0이 되면 자동으로 `CartProvider`의 상품 삭제 함수를 호출하여 목록에서 제거합니다.
    -   **(추후 고려)** 최대 주문 수량(재고) 제한을 고려하여, `+` 버튼 클릭 시 재고를 초과하면 수량 증가를 막는 로직을 추가합니다.

#### **2-4. `<CartSummary />` (결제 요약 정보)**

* **주요 역할**: 상품 금액, 배송비, 총 금액을 계산하고 표시하며, 결제하기 버튼을 렌더링합니다.
* **사용 컴포넌트**: `PriceRow`, `CheckoutButton`
* **세부 작업**:
    1.  `CartProvider`에서 장바구니 상품 목록을 가져옵니다.
    2.  모든 상품의 가격과 수량을 곱하여 **상품 금액**을 계산합니다.
    3.  계산된 상품 금액이 100,000원 이상일 경우, 배송비를 `0원`으로 설정합니다.
    4.  그 외의 경우, 배송비를 `3,000원`으로 설정합니다.
    5.  상품 금액과 배송비를 합산하여 **총 금액**을 계산합니다.
    6.  "결제하기" 버튼 클릭 시, 라우팅을 통해 카드 선택 페이지로 이동합니다.
* **예외처리**:
    -   장바구니가 비어있을 경우, 모든 금액을 `0원`으로 표시하고 "결제하기" 버튼을 비활성화하거나 숨길 수 있습니다.

#### **2-5. `CartProvider` (장바구니 상태 관리)**

* **주요 역할**: 장바구니에 담긴 상품 목록의 상태를 관리하고, 모든 컴포넌트에서 접근할 수 있는 인터페이스를 제공합니다.
* **세부 작업**:
    1.  `useState`를 사용하여 상품 객체들의 배열을 상태로 관리합니다.
    2.  `addItem(product)`, `removeItem(productId)`, `updateQuantity(productId, newQuantity)`와 같은 상태 변경 함수를 정의합니다.
    3.  `updateQuantity` 함수는 `newQuantity`가 0일 경우 자동으로 `removeItem`을 호출하도록 로직을 포함합니다.
* **예외처리**:
    -   `addItem` 시 이미 장바구니에 있는 상품이라면 수량만 증가시키고, 없는 상품이라면 새로 추가하는 로직을 구현합니다.
