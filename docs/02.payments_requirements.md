## 📝 고객 요구사항 정리
1. 카드 목록 조회 및 등록 기능 제공: 

사용자가 보유한 카드를 목록으로 확인할 수 있어야 함.
새로운 카드를 추가(등록)할 수 있는 기능을 제공해야 함.
등록된 카드는 마스킹된 형태로 표시되어야 함.

2. 카드정보 입력 기능:

카드 번호 입력:
  숫자만 입력 가능해야 하며, 카드사별 유효성 검사 및 자릿수 제한이 필요함.
  입력 시 xxxx - xxxx - xxxx - xxxx와 같은 형태로 자동 포맷팅되어야 함.
만료일 입력:
  MM/YY 형식으로 입력 가능해야 함.
  유효한 월(1-12) 및 현재 날짜 이후의 연도만 허용해야 함.
카드 소유자 이름 입력:
  최대 30자까지 입력 가능해야 함.
  입력 중 실시간으로 (현재글자수/30) 형태로 글자 수를 표시해야 함.
보안 코드 (CVC/CVV) 입력:
  3-4자리 숫자로 입력 가능해야 함.
  입력 필드 옆에 물음표 아이콘을 통해 도움말 툴팁 또는 팝업을 제공해야 함.
카드 비밀번호 입력:
  4자리 숫자로 입력 가능해야 함.
  앞 2자리만 사용자 입력으로 받고, 뒤 2자리는 시안과 같이 미리 마스킹된 상태(•)로 표시되어야 함.

3. 민감 정보 보안 처리 (UI 마스킹): 
 
  카드 번호: 입력 완료 시 뒤에서 8자리가 ••••-•••• 형태로 마스킹되어야 함.
  보안 코드 (CVC/CVV): 입력 시 ••• 형태로 마스킹되어야 함.
  카드 비밀번호: 입력 시 • 형태로 마스킹되어야 함.
  마스킹에 사용되는 점(.)의 형태 (width: 7px; height: 7px;)를 유지해야 함.

4. "작성 완료" 버튼:

카드 추가 화면에서 모든 필수 입력 필드*카드 번호, 만료일, 카드 소유자 이름, 보안 코드, 카드 비밀번호)가 유효하게 입력되었을 떄만 "작성 완료" 버튼이 활성화되어야 함.(활성화 전에는 비활성화 상태 또는 미표시)

5. "이 카드로 결제하기" 버튼:

등록된 카드를 선택하여 진행할 수 있는 버튼을 제공해야 함.
버튼 스타일은 type="primary", size="medium", icon="off" 속성을 따름.

6. 화면 간 네비게이션:
나의 카드 목록 화면에서 카드 추가 버튼 클릭 시 카드 추가 화면으로 이동해야 함.
카드 추가 화면에서 "작성 완료" 버튼 클릭 시 카드 등록 처리 후 나의 카드 목록 화면(등록된 카드 표시)으로 이동해야 함.
보유 카드 목록 화면에서 추가 카드 등록 버튼 클릭 시 다시 카드 추가 화면으로 이동해야 함.
각화면 상단의 '뒤로 가기' 및 '닫기' 버튼은 해당 화면을 닫거나 이전 화면으로 이동하는 기능을 수행해야 함.

7. 배포 URL:
최종 서비스는 고객사가 제공하는 특정 URL로 배포되어야 함.

  
## 📋 기능 목록
본 결제 모듈은 React 기반의 컴포넌트 아키텍처로 구현될 예정이며, 주요 기능별 컴포넌트 및 필요한 기능들은 다음과 같습니다.

<App /> (전체 애플리케이션 라우팅 및 상태 관리)
  
    /cards 경로: 나의 카드 목록 화면 렌더링
  
    /cards/add 경로: 카드 추가 화면 렌더링

  
<CardListScreen /> (나의 카드 목록 화면)

  UI 요소:
 
    "나의 카드 목록" 헤더
 
    닫기 버튼 (X)
  
    등록된 카드 컴포넌트 (<RegisteredCard /> - 옵션)
 
    카드 추가 버튼 (<AddCardButton />)


기능:

    등록된 카드 정보 표시 (마스킹 처리)
      
    카드 추가 버튼 클릭 시 카드 추가 화면으로 이동 (/cards/add)


<AddCardScreen /> (카드 추가 화면)

UI 요소:

    "카드 추가" 헤더
  
    뒤로 가기 버튼 (<)
    
    닫기 버튼 (X)
  
    카드 이미지 미리보기 (<CardPreview />)
  
    카드 번호 입력 필드 (<CardNumberInput />)
  
    만료일 입력 필드 (<ExpiryDateInput />)
  
    카드 소유자 이름 입력 필드 (<CardOwnerNameInput />)
  
    보안 코드(CVC/CVV) 입력 필드 (<CVCInput />)
  
    카드 비밀번호 입력 필드 (<PasswordInput />)
  
    작성 완료 버튼 (<SubmitButton />)


기능:

    각 입력 필드로부터 값 입력 받기 및 유효성 검사
  
    입력 값에 따른 UI 마스킹 처리 (카드 번호, CVC, 비밀번호)
  
    카드 소유자 이름 글자 수 카운트 및 제한
  
    모든 필수 입력 완료 시 작성 완료 버튼 활성화/비활성화 제어
  
    작성 완료 버튼 클릭 시 카드 등록 로직 실행 및 나의 카드 목록 화면으로 이동
  
  

<RegisteredCard /> (등록된 카드 개별 컴포넌트)

UI 요소:

    등록된 카드의 시각적 표현 (디자인 시안 기반)
  
    마스킹된 카드 번호, 소유자 이름, 만료일 표시
  
    "이 카드로 결제하기" 버튼 (<PayButton />)


기능:

    "이 카드로 결제하기" 버튼 클릭 시 결제 로직 트리거

<AddCardButton /> (새로운 카드 추가 버튼)

기능:

    클릭 시 카드 추가 화면(AddCardScreen)으로 이동
